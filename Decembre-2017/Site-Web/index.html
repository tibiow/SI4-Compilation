Compilation

Objectifs

Les objectifs du cours de compilation sont:

  • Assimiler les théories, méthodes et techniques qui permettent de décrire et concevoir des langages.
  • Comprendre les principes de conception des langages de programmation et les transformations de programmes réalisées par les compilateurs.
  • Comprendre finement les différents aspects des langages de programmation, en particulier les vérifications et transformations effectuées par les compilateurs.
  • Assimiler les techniques qui permettent l'écriture de compilateurs.

Afin de mettre en pratique les principes et les outils étudiés les dernières séances TDs sont consacrés à l'adjonction d'un certain nombre d'extensions sur un compilateur pour un petit langage de programmation à objets.

Enseignant: Erick Gallesio

Organisation du cours

Le cours de compilation est formé des parties suivantes:

  • Introduction à la compilation. Cette partie montre les composants principaux d'un compilateur.

  • Analyse lexicale. C'est la première phase d'un compilateur. On se contentera de rappeler ici les principes d'un analyseur lexical, puisque l'outil lex est étudié en première année du cycle ingénieur dans le cours «Langages Formels et Automates».

  • Analyse syntaxique. C'est la seconde phase du processus de compilation. On étudiera différents type de grammaires et différentes façons de les analyser (analyses descendantes et ascendantes).
    On se servira ensuite de yacc (ou sa variante GNU: bison) le générateur d'analyseurs syntaxiques ascendants pour analyser des petits langages.
    Cette partie se termine par la réalisation d'une calculatrice évoluée avec production de code pour différentes parties arrières.

  • Réalisation d'un compilateur pour un petit langage objet de type C++/java. Le compilateur construit dans cette partie produit, après analyse du texte source, un programme C qui peut ensuite être compilé avec un compilateur C standard.

Plan du cours

  • Introduction à la compilation
  • Analyse lexicale (ce cours correspond à ce qui est fait en première année dans le cadre de LFA)
  • Analyse syntaxique
  • Analyse descendante
  • Analyse ascendante
  • Yacc/Bison
  • Implémentation de l'arbre syntaxique
  • Toy un mini langage objet
  • Analyse sémantique statique
  • Implémentation des objets

Travaux Dirigés

Les feuilles de TD associées à ce module:

  • Feuille 1: Analyse lexicale avec lex/flex (Rappels)
  • Feuille 2: Grammaires, Analyse descendante
  • Feuille 3: Grammaires, Analyse ascendante
  • Feuille 4: Utilisation de yacc sur des grammaires simples
  • Feuille 5: Implémentation d'un mini-langage
  • Feuille 6: Extensions du langage Toy-base
  • Feuille 7: Extensions du langage Toy-base (suite)
  • Feuille 7: Extensions du langage Toy-full

Pré-requis

  • un pré-requis évident pour ce cours est l'enseignement Langages Formels et Automates de SI3.
  • Une bonne connaissance du langage C est aussi nécessaire pour les TDs.

Quelques références bibliographiques

  • Compilateurs : principes, techniques et outils - 2e édition de Alfred Aho, Monica Lam, Ravi Sethi et Jeffrey Ullman. Ed Pearsons (2007). Ce livre (appelé le dragon book) est la bible en matière de compilation. C'est un peu théorique et parfois un peu daté, mais ça reste incontournable.

Mode d'évaluation

La note finale de ce module est constituée de 3 notes:

  • Une note de TD construite à partir des exercices rendus pendant à la fin de certaines séances de TD. Cette note comptera pour 30% de la note finale.

  • Une épreuve sur table à mi-parcours du semestre qui comptera pour 30% de la note finale. Le programme de l'épreuve portera sur la première partie du cours.

  • Une épreuve sur machine à la fin de l'enseignement qui portera sur la totalité des notions abordées dans le module. Cette épreuve comptera pour 40% de la note finale.
