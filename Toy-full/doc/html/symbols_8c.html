Detailed Description

The module SYMBOLS is in charge of implementing the various symbol tables needed by the compiler. We have two categories of tables:

  • tables implementing scopes: a new table is created when we enter a new scope (entering a function or a block). These tables are linked so that we can search a symbol (denoting a variable, a function, a class, ...)from the inner scope to the global one.
  • tables in charge of class members management. We have one table for each class declared in a program. Here again tables are linked so that we can access, from a class, the members of its super class.
