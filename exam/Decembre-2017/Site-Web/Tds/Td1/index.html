Feuille 1: Analyse lexicale avec Lex / Flex.

Cette feuille est une feuille de révision sur l'outil lex que vous aviez vu dans le module LFA de l'an dernier. On utilisera ici l'outil lex (ou flex) qui permet de construire automatiquement un analyseur lexical en C à partir d'un fichier de spécification.

Sujet

Le sujet de la feuille de TD est ici.

Corrigés

Calculatrice

Le corrigé de la calculatrice est simple. Il est donné ci-dessous 

/*
 *  calc.l              -- Un lexical simple pour une calculette avec variables
*/

%{
#include <stdio.h>
#include "calc.h"
%}

%option noyywrap

%%

[0-9]+                  { yylval.val = atoi(yytext); return NUMBER; }
[a-zA-Z]+               { yylval.var = strdup(yytext); return IDENT; }

"+"                     { return PLUS;  }
"-"                     { return MINUS; }
"*"                     { return MULT;  }
"/"                     { return DIV;   }
"("                     { return OPEN;  }
")"                     { return CLOSE; }
"="                     { return EQUAL; }
\n                      { return EOL; }


[ \t]                   { }
.                       { fprintf(stderr, "Unexpected character %c (%d)\n",
                                          *yytext, *yytext); }
%%

Notes:

  • Il faut inclure kle fichier calc.h qui contient les définitions de constantes pour les token, ainsi que la déclaration de la variable yylval.
  • Les seuls cas ou il faut faire attention sont les identificateurs et les entiers: Dans ce cas, il faut remplir le bon champ de l'union yylval abvant de renvoyer le type token rencontré.

  • Pour compiler (le source lex étant dans le fichier calc.lex.c

    $ lex  -o calc.lex.c calc.l
$ gcc -Wall -std=gnu99 calc.c calc.lex.c -o calc

Espaces et langages de programmation

Avant de commencer, il faut savoir que l'expression - - a est valide en C: elle correspond à la valeur (- (- a)), c'est à dire a. Par contre, l'expression --a sans espace entre les deux tirets correspond à la pré-décrémentation de la variable a.

Par conséquent, * l'expression - -1 est valide et correspond à l'entier +1, alors que * l'expression --1 n'est pas valide puisque on ne peut pas décrémenter une constante.

La table demandée est donc:

Expression Valeur x y Valeur sans espace x y
x - 1 0 1 2 0 1 2
x - -1 -2 1 2 erreur
x - - 1 2 1 2 erreur
x - - - 1 0 1 2 0 0 2
x - y -1 1 2 -1 1 2
x - -y 3 1 2 erreur
x - - y 3 1 2 erreur
x ---y -1 0 2 -1 0 2
-y -2 1 2 -2 1 2
- -y 2 1 2 -2 1 2
--y 1 1 1 1 1 1
x -- - -- y 0 0 1 erreur
x-- + --y 2 0 1 2 0 1

Comme on peut le voir, les espaces peuvent donc être significatifs, même si un analyseur lexical ne renvoie en général pas de lexème correspondant à un espace en tant que tel.

Minifier

Dans un premier temps, nous allons simplement enlever les commentaires de nos programmes. Un première idée pourrait consister à dire qu'ils sont purement et simplement effacés en sortie. Malheusement ce n'est pas complètement sûr; prenons le morceau de programme C suivant

   x = 3-/* Un commentaire */-3;

on obtiendrait c x = 3--3; qui est incorrect, comme nous l'avons vu dans l'exercice précédent.

Par conséquent, les commentaires classiques seront remplacés par un espace et les commentaire à la C++ seront remplacé par un saut de ligne.

Par ailleurs, les directives du pré-processeur seront laissée telles quelles (en fait c'est un peu plus compliqué, car il faudrait aussi traiter les lignes suites qui se terminent par un \ en fin de ligne; on ne le fera pas ici).

Bien sûr les chaînes ne doivent pas être modifiées du tout. Elles seront recopiées telles quelles.

Quant aux suites d'espaces, elle seront remplacée par un espace unique.

Nous obtenons l'analyseur suivant:

/*
// Un «minifier» pour C
*/

%{
#include <ctype.h>
%}

%option noyywrap
%x COMMENT

space   [ \t\n]

%%
 /* strings */
\"([^\"]|\\.)*\"                { printf("%s", yytext); }

 /* Commentaires classiques */
{space}*"/*"                    { BEGIN COMMENT; }
<COMMENT>"*/"{space}*           { BEGIN INITIAL; putchar(' ' );}
<COMMENT>.|\n                   { }

  /* Commentaires  C++ */
{space}*"//".*$                 { putchar('\n');}

 /* Directives  FAUX si on a 1 ligne suite */
"#".*                           { printf("%s\n", yytext);  }

  /* Suite d'espaces => un seul espace  */
{space}+                       putchar(' ');

  /* Les autres caractères */
.                               ECHO;

%%

int main() {
    yylex();
    putchar('\n');      // to avoid to be "crunched" by a prompt
}
Note:
Le remplacement d'un commentaire par un espace ou un newline, n'est pas une très bonne idée. En effet si deux commentaire se suivent, il seront remplacés deux fois par un espace en sortie (et donc occuperont deux caractères. Une meilleure solution consiste à utiliser la macro unput qui permet de remettre un caractère dans le flux d'entrée de flex. Ainsi, deux commentaires successifs seront remplacé par un seul espace. Yeah!!

En faisant tourner l'analyseur précédent sur le programme hello.c, on obtient:

 #include <stdio.h>
 char *w = "world"; int main() { printf("Hello, %s.\n", w); return 0; }

C'est déjà pas mal, mais on peut faire mieux: il reste des espaces que l'on peut facilement éliminer. En effet, si on excepte + et - qui sont problématiques, on peut supprimer les espaces autour des séparateurs (c'est à dire ,, =, !, ...).

Il suffit donc de rajouter au programme précédent 

 /* opérateurs & séparateurs qui ne necessitent pas d'espaces à gauche et à droite. */
separ [\"\'=!*<>,;:{}\[\]()*~|&]

.....

  /* suppression des espaces avant et après  les séparateurs */
{space}*{separ}{space}*         { for (char *s=yytext; *s; s++)
                                  if (!isspace(*s)) putchar(*s);
                                }

En faisant tourner cette nouvelle version de l'analyseur sur le programme hello.c, on obtient:

 #include <stdio.h>
 char*w="world";int main(){printf("Hello, %s.\n",w);return 0;}

C'est pas top mal sur cet exemple, mais ce code n'est pas optimal, car il laisse encore des espaces inutiles (typiquement si on a des commentaires en fin de ligne). Noter qu'en mettant la sortie du minifier en pipe avec lui même, ceux-ci seront supprimés....
